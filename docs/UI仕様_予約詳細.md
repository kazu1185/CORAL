# 予約詳細画面 UI仕様書

UIモック（pms_reservation_detail.html）を参考にした画面仕様。

---

## 1. 画面の役割

PMSの中核画面。1画面で全操作を完結する。

- CI/CO処理
- 延泊・部屋変更・キャンセル
- 領収書・請求書の発行
- ゲスト紐付け（名寄せ）
- 入金記録
- 売上明細の確認（連結予約の場合はグループ全体）

CI/CO操作は専用画面に遷移せず、この画面の中で完結する。

---

## 2. 画面への入口

```
① ダッシュボード CI/COリスト → 行クリック → 予約詳細
② ルームインジケーター → カードクリック → 予約詳細
③ アサインボード → バークリック → 予約詳細
④ 予約一覧 → 行クリック → ドロワー → [予約詳細画面へ]
```

## 4. トップバー

1行で予約の基本情報をすべて表示。

| 要素 | 表示 |
|------|------|
| 戻るボタン | ← 戻る |
| 部屋番号 | 309（大きい数字） |
| 部屋タイプ | ツイン |
| ステータスバッジ | CI済（緑ピル）/ 確定（青ピル）等 |
| CI日・CO日 | CI: 4/09(水) │ CO: 4/11(金) |
| 泊数 | 2泊 |
| 連結表示 | 🔗連結5泊（紫バッジ、連結時のみ） |
| OTA | カラードット + OTA名 |
| 予約番号 | モノスペース |

---

## 5. アクションバー

### 5-1. ボタン構成

```
[CI] [CO] [延泊] [部屋変更] [キャンセル]  │  [領収書] [請求書]
```

セパレータ（│）で操作系と帳票系を視覚的に分離。

### 5-2. ステータス別のボタン有効/無効

| ステータス | CI | CO | 延泊 | 部屋変更 | キャンセル | 領収書 | 請求書 |
|-----------|----|----|------|---------|----------|--------|--------|
| confirmed | ✅有効 | 無効 | 無効 | 有効 | 有効 | 有効 | 有効 |
| checked_in | 無効 | ✅有効（緑） | 有効 | 有効 | 無効 | 有効 | 有効 |
| checked_out | 無効 | 無効 | 無効 | 無効 | 無効 | 有効 | 有効 |
| cancelled / no_show | 全無効 | | | | | 有効 | 有効 |

領収書・請求書ボタンは全ステータスで常に有効（CO後の再発行、キャンセル料の領収書が必要なため）。

## 6. アラートバー

トップバーとアクションバーの下に表示。状態に応じて内容が変わる。

| 状態 | 色 | テキスト |
|------|-----|---------|
| 子供料金未入力 | 青（info） | ℹ️ 子供料金が未入力です（均等割りで宿泊税計算） |
| ゲスト未確定 | 黄（warn） | ⚠️ ゲストが未確定です。ゲストカードをクリックして候補から選択してください。 |
| 部屋未アサイン | 赤（error） | 🔴 部屋が未割当です。先にアサインしてください。 |

## 7. 左カラム（260px固定）

### 7-1. ゲストカード

- ゲスト名（大きい文字）+ カナ / ローマ字
- バッジ群（⭐VIP / 🔁×N / 🔗連結）
- **クリック可能** → ゲスト詳細パネルが開く（ホバーでグレー背景 + ▶マーク）
- pending時はタイトル行の右端に「⚠ 未確定」赤バッジが表示

### 7-2. 属性セクション

| 項目 | 表示例 |
|------|-------|
| 宿泊人数 | 大人2名 子供1名 |
| 宿泊回数 | 3回（pending時は「—」） |
| 国籍 | 🇯🇵 日本 |
| 支払方法 | 現地（現金） |
| 領収書宛名 | 對馬 晋 |

### 7-3. ゲストメモ（永続）

- 赤枠の警告ボックスで目立つ表示
- guest_notesが空でなければ表示、空なら非表示
- pending時は非表示（ゲスト未紐付けのためメモがない）

### 7-4. 予約メモ（今回の滞在のみ）

- テキストエリア入力欄
- reservation_notesに保存

## 8. 中央カラム（可変幅）— 売上明細

### 8-1. 連結予約の場合

連結グループ全体を1つの予約として通しで表示。

- タイトル: 「💰 売上明細（連結N泊分）」
- OTAグループヘッダー行: グレー背景で「🔴 じゃらん 8RAX3GPY（この予約）」等
- 「この予約」の行は黄色背景でハイライト
- 各グループの小計行
- 連結全体の合計（小計 + 消費税 + 宿泊税）

### 8-2. 単独予約の場合

通常の1泊1行テーブル + 合計。

### 8-3. テーブル列構成

| 列 | 内容 |
|----|------|
| 日付 | 宿泊日 |
| 摘要 | プラン名 |
| OTA | カラードット（連結時のみ列表示） |
| 単価 | 税抜金額 |
| 消費税 | 参考値（正式な税額は合計に対して計算） |
| 宿泊税 | 1泊あたりの宿泊税 |

### 8-4. 入金セクション

- 「入金（連結全体）」または「入金」ヘッダー（緑背景）
- 各入金行: 日付 + 決済方法 + OTAドット（連結時）+ 金額
- **「＋ 入金を記録」ボタン**（青い破線ボタン）
  - クリック → 入金記録ダイアログ（金額・決済方法プルダウン・入金日）
  - 決済方法: 現金 / クレジットカード / 電子マネー / QRコード決済 / OTA事前決済 / 法人売掛
- 残額表示: ¥0は緑、未払いは赤

## 10. ゲスト詳細パネル（オーバーレイ）

ゲストカードをクリックで開く。matched/pendingで内容が切り替わる。

### 10-1. matched時（紐付け済み）

「👤 ゲスト詳細」パネル：

- 氏名（大きい文字）+ カナ / ローマ字 + バッジ
- 基本情報（国籍・メール・電話・宿泊回数・初回/最終宿泊日）
- 永続メモ（全文）
- 宿泊履歴（日付・OTAドット・部屋・金額）
- アクションボタン: [閉じる] [名前を修正] [姓の変更（旧姓保存）] [他のゲストと統合]

### 10-2. pending時（ゲスト未確定）— 名寄せUI

「👤 ゲスト確認」パネル：

**TL原本データ:**
- OTA名（書き換え不可）
- チャネル名
- 表示名（自動変換結果、編集可能）

**ゲスト候補リスト:**

各候補カードに以下を表示：
- 候補名（カナ + 漢字）+ マッチタイプバッジ（カナ一致 / ローマ字類似 / 旧姓一致）
- サマリー（前回宿泊日・宿泊回数）
- ゲストメモの要約（あれば）
- **2ボタン横並び:**
  - [詳細を見る ▼] → アコーディオン展開
  - [この方です ✓] → 紐付け確定

**「詳細を見る」で展開される情報:**
- 連絡先（メール・電話・国籍）
- 永続メモ（全文）
- 宿泊履歴（全件、日付・OTAドット・部屋・金額）

**候補リスト下部:**
- [＋ 新規ゲストとして登録] ボタン（破線）

**設計判断:** 候補のサマリー情報だけでは同姓同名の別人かどうか判断できないケースがあるため、「詳細を見る」で宿泊履歴・連絡先・メモ全文を確認してから確定できるようにした。

## 11. 確認ダイアログ

### 11-1. チェックアウト確認

- ゲスト名 + 部屋番号
- 精算済み金額
- 領収書発行チェック（宛名・但書入力付き）
- guest_notesがあれば警告表示（赤枠）

### 11-2. 延泊

- 現在の日程 → 延泊後の日程
- 追加料金
- 精算タイミング選択（今すぐ / CO時にまとめて）

### 11-3. 領収書発行

- 合計金額
- 宛名・但書・発行日
- 分割発行への案内

### 11-4. 入金記録

- 売上合計 / 入金済み / 残額のサマリー
- 入金額入力
- 決済方法プルダウン
- 入金日
