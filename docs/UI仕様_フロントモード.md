# UI仕様 — フロントモード（iPad 13インチ対面カウンター業務）

作成: 2026-07-24（設計: Claude Fable 5）
親計画: `docs/iPad対応_タブレットUI計画_2026-07.md`
**ビジュアルの正: `mock/pms_front_mode.html`**（iPadのSafariで開いて確認できる。本仕様書と食い違う場合はモックを正とし、本仕様書を直す）

実装者（Opus 4.8）への指示: 本仕様は「解釈の余地を残さない」ことを目的に書いている。寸法・色・状態は本仕様とモックに従い、独自判断で変えない。変えたい場合はユーザーに確認する。

## 1. デザイン原則（Patinaのタブレット拡張）

Patinaデザイン言語（`docs/UI刷新_Patina_2026-07.md`）を完全に継承した上で、以下を加える。

1. **1画面1目的**。フロント業務は接客中に行うため、1画面に複数の判断を求めない
2. **44pt最小・主要操作は56pt**。タップターゲットは Apple HIG の44ptを最小とし、業務の主動線（カード・主ボタン）は56pt以上
3. **視距離が遠い**。カウンターに置いたiPadは腕1本分遠い。本文16px・見出し20px以上・金額は最大級（40px）
4. **色はPatinaトークンのみ**（variables.css の値）。ハードコード禁止。OTA・ステータスの識別色は既存共通CSSを使う
5. **静かな成功、明確な失敗**。成功はチェックマークの短い演出（800ms）で流し、失敗・ブロックは面色＋文言でハッキリ止める

## 2. フロントモード専用トークン（variables.css への追加分）

既存トークンに加え、`--front-*` プレフィックスで追加する（既存画面に影響させないため）。

```css
/* フロントモード（タブレット）追加トークン */
--front-header-h: 56px;      /* ヘッダー高 */
--front-tabbar-h: 64px;      /* タブバー高 */
--front-target-min: 44px;    /* 最小タップターゲット */
--front-target-lg: 56px;     /* 主要ボタン高 */
--front-target-xl: 64px;     /* 最重要CTA高（CI実行/CO実行/会計） */
--front-font-body: 16px;     /* 本文（iOS入力ズーム回避を兼ねる） */
--front-font-title: 20px;    /* カード見出し・画面タイトル */
--front-font-hero: 40px;     /* 残額・合計などの最大表示 */
--front-gap: 16px;           /* 基本余白（8pxグリッド: 8/12/16/24/32） */
```

## 3. レイアウト骨格（FrontLayout）

```
┌──────────────────────────────────────────────┐
│ ヘッダー 56px: [珊瑚アイコン CORAL PMS] [7/24(金) 14:32] [山田] [管理画面↗] │
├──────────────────────────────────────────────┤
│ タブバー 64px: [本日のチェックイン◦3] [本日のチェックアウト◦1] [物販] [部屋状況] │
├──────────────────────────────────────────────┤
│                                              │
│ コンテンツ（スクロール領域）                        │
│ 最大幅1280px・中央寄せ・左右パディング24px            │
│                                              │
└──────────────────────────────────────────────┘
```

- ルート要素: `height: 100dvh`（100vhフォールバック併記）。ヘッダー・タブバーは固定、コンテンツのみスクロール
- セーフエリア: ヘッダーに `padding-top: env(safe-area-inset-top)`、コンテンツ最下部に `padding-bottom: calc(24px + env(safe-area-inset-bottom))`
- 背景 `--bg-main`、面は `--bg-card`、影 `--shadow-sm`。サイドバーは無い
- **ヘッダー**: 左=珊瑚アイコン+「CORAL PMS」ワードマーク（`coral-icon-dark.svg`。JSXから`<img>`参照 = 規約 #22）、中央=日付時刻（1分毎更新）、右=ログインスタッフ名チップ（タップでログアウトメニュー）＋「管理画面」テキストリンク（`--text-secondary`・14px。目立たせない）
- **タブバー**: 4タブ等幅ではなく内容幅+余白。各タブ = アイコン20px + ラベル16px/500、高さ64px・横パディング24px
  - アクティブ: 文字 `--accent-blue`・700、下ボーダー3px `--accent-blue`、面 `--accent-blue-soft`
  - 非アクティブ: `--text-secondary`。press時 `--bg-hover`
  - CI/COタブには**残件バッジ**（未処理件数。`--accent-red`地・白文字・最小20px円）。0件なら非表示
- タップフィードバック共通: `:active` で `transform: scale(0.98)` + 面色変化、150ms。`-webkit-tap-highlight-color: transparent`

## 4. 画面仕様

### 4.1 ログイン（PINパッド）

共用端末で最速・確実にログインする画面。認証APIは既存（スタッフ名+PIN）のまま。

- 画面中央にカード（幅 720px・角丸 `--radius-lg`・`--shadow-md`）を横2ペイン構成で置く。上部に珊瑚ロゴ+「CORAL PMS」
- **左ペイン: スタッフ選択**（タイプさせない）
  - アクティブスタッフを縦リストのボタンで表示。各 高さ56px・幅100%・16px/500・角丸 `--radius-md`
  - 選択中: `--accent-blue-soft` 面 + `--accent-blue` 文字 + 左ボーダー3px
  - 7名以上はスクロール（フロント6名想定なので通常収まる）
- **右ペイン: PIN入力**
  - PINドット表示: ●4個（入力済み=`--accent-blue`塗り、未入力=`--border-strong`輪郭のみ）、直径14px・間隔16px
  - テンキー: 3列×4行（1-9、最下段=空/0/⌫）。キー 88×64px・間隔12px・20px/500・角丸 `--radius-md`・`--bg-inset`面・press時 `--accent-blue-soft`
  - 4桁入力で**自動送信**（ログインボタン不要）。失敗時はドットを赤く2回シェイク＋「PINが違います」（`--accent-red`・14px）→ 自動クリア
  - 物理キーボード入力も並行して受け付ける（PC・外付けキーボード互換）
- ログイン成功 → フロント端末フラグがあれば `/front` へ

### 4.2 本日ボード — チェックインタブ / チェックアウトタブ

構造は両タブ共通。データは既存 `GET /dashboard`（`checkin_list` / `checkout_list`）+ 5〜10秒ポーリング。

- **サマリー行**（コンテンツ最上部・カード外）: 「本日のチェックイン **8件** ・ 完了 **3件**」16px + 細プログレスバー（高さ4px・`--accent-blue`・地 `--border-light`）。右端にゲスト名検索フィールド（幅280px・高さ44px・虫眼鏡アイコン）
- **予約カード**: 縦1列リスト（1行1予約。2カラムにしない — 視線が往復せず、接客中に指を滑らせるだけで探せる）
  - 高さ 88px・`--bg-card`・角丸 `--radius-md`・`--shadow-sm`・下マージン12px・横パディング20px
  - 左から:
    1. **部屋番号チップ**: 64×56px・`--bg-inset`面・角丸 `--radius-sm`・20px/700。未アサイン時は `--accent-red-soft` 相当面（rgba(179,54,44,0.08)）に「未」表示
    2. **ゲスト名ブロック**: 名前 20px/600 `--text-primary`（表示名。原本tl_*は書き換えない=規約 #12）、下段にメタ 14px `--text-secondary`「2名 ・ 3泊 ・ ツイン ・ 朝食付」
    3. **OTAバッジ**: 既存 `styles/ota-badge.css` をそのまま使用（サイズも既存のまま。識別性優先）
    4. 右端: **ステータス**。未処理=「チェックイン →」の矢印付きテキスト16px/`--accent-blue`、処理済み=既存ステータスバッジ（CI済/CO済）＋カード全体の不透明度を0.55に落として下方にソート
  - アラート系（ゲスト未確定等）はカード左に3pxの `--accent-red` ボーダー＋名前の下にアラート文言 13px 赤
  - タップ: カード全体がターゲット。press時 `--bg-hover` + scale(0.98) → CI確認/CO精算画面へ
- **空状態**: 中央に珊瑚アイコン（淡色）+「本日のチェックインはすべて完了しました 🎉」相当の文言 16px `--text-muted`
- **ローディング**: 初回のみスケルトンカード3枚（`--bg-inset` のシマー）。ポーリング更新では画面を暗転させない

### 4.3 チェックイン確認画面

`/front/checkin/:id`。1画面で「予約を確認して、CIを実行する」だけ。

- **画面ヘッダー行**: 左に戻るボタン（44×44px・←アイコン・`--bg-inset`円形）+「チェックイン」20px/700 + 右にゲスト名（呼び出し用に大きく 20px）
- **本体: 横2ペイン**（landscape。gap 24px）
  - **左ペイン「予約内容」カード**（幅 55%）: 定義リスト形式（ラベル13px `--text-muted` / 値16px、行高44px、行間ボーダー `--border-light`）
    - 日程（IN/OUT・泊数）、部屋（**部屋番号を20px/700**・部屋タイプ）、プラン名、人数、チャネル（OTAバッジ）、予約番号、料金合計・入金済み・**残額**（残額>0なら `--accent-red` で強調）
    - TL原本と変更がある項目は既存予約詳細と同じ注記方式
  - **右ペイン「ゲスト」カード**（幅 45%）:
    - ゲスト名・フリガナ・電話・住所（既存ゲストデータ表示）
    - **パスポート枠**（外国籍ゲストのみ表示）: 撮影済みならサムネイル（高さ120px・角丸）、未撮影なら破線枠 + 「📷 パスポートを撮影」ボタン（高さ56px・セカンダリスタイル）→ `<input capture>` でカメラ起動
    - 備考・アラート（ゲスト未確定なら赤帯+「PCで顧客確定が必要です」）
- **下部固定アクションバー**（高さ 88px・`--bg-card`・上ボーダー・左右パディング24px・セーフエリア加算）
  - 右端: **「チェックインする」プライマリボタン**（高さ64px・最小幅320px・18px/700・`--accent-blue`地/白文字・角丸 `--radius-md`）
  - 押下 → `useConfirm` の確認（規約 #14、ダイアログのボタンは高さ52pxに拡大）→ API `POST /reservations/:id/checkin`（updated_at同送=規約 #16）
  - 成功 → 全画面オーバーレイに ✓ チェックマーク演出800ms →本日ボードへ自動復帰
  - **ブロック状態**: 部屋未アサイン時はボタンを `--text-muted` 地で無効化し、バー左側に理由を明示「⚠ 部屋が未アサインです。PCのアサインボードで割り当ててください」（16px・`--accent-red`）
  - 409（楽観ロック）→ 「他の端末で更新されました。再読み込みします」→ 再取得
- **グループ予約**: 左ペイン下に子予約リスト（部屋番号+ゲスト名の行・各44px）を表示し、アクションボタンを「グループ全体をチェックイン（n室）」に差し替え（groupCheckin）

### 4.4 チェックアウト精算画面

`/front/checkout/:id`。フロー: 明細確認 → （不足あれば入金登録）→ 領収書 → CO実行。

- 画面ヘッダー行はCI画面と同型（タイトル「チェックアウト」）
- **左ペイン「利用明細」カード**（幅 55%）
  - 明細行: 行高48px・日付13px muted・摘要16px・金額16px右寄せタブラー数字（`font-variant-numeric: tabular-nums`）
  - goods行は鍵アイコンのまま編集不可（規約 #25）。**明細の追加・編集はこの画面ではさせない**（複雑精算はPCの予約詳細へ。カード下部に「明細の修正はPCの予約詳細から」13px muted）
  - 宿泊税・消費税の区分は既存領収書ロジック準拠の表示
- **右ペイン「精算」カード**（幅 45%・上から順に）
  1. 金額サマリー（`--bg-inset`面・パディング20px）: 合計16px / 入金済み16px / 罫線 / **残額 40px/700**。残額0円=`--accent-green`「精算済み」、>0=`--text-primary`（赤にしない。請求は日常動作）
  2. **入金登録**（残額>0時のみ活性）: 支払方法ボタン群（マスタ駆動: 現金/カード/…。各 高さ56px・グリッド2列・選択中は `--accent-blue-soft`+ボーダー`--accent-blue`）+ 金額入力（デフォルト=残額・タブラー20px・右寄せ）+「入金を登録」セカンダリボタン56px
  3. **「領収書を発行」ボタン**（高さ56px・セカンダリ: `--bg-card`地・`--accent-blue`文字・1pxボーダー`--accent-blue`）→ 宛名入力（`useConfirm` の prompt）→ PDF表示（iPadでの表示方式はPhase 3のスパイク検証結果に従う）
- **下部固定アクションバー**: 「チェックアウトする」プライマリ64px。**残額>0のときは確認ダイアログで「残額 ¥12,340 がありますがチェックアウトしますか？」と明示的に警告**（ブロックはしない。後精算の運用があり得るため）。グループはCI画面と同様に一括CO
- 成功演出・409処理はCI画面と同一

### 4.5 物販タブ（即売POS）

既存 ProductSalesPage の機能を**フロントモード内タブとして再構成**する（画面の新規実装。API・売上経路ルールは既存のまま=規約 #25/#27。即売=reservation_id NULL）。

- **横2ペイン**: 左=商品グリッド（残り幅）、右=カート（固定幅 360px）
- **商品グリッド**: カード 幅約160px（`repeat(auto-fill, minmax(150px, 1fr))`）・高さ112px・角丸 `--radius-md`
  - 商品名16px/500（2行まで）+ 価格15px/600 タブラー。カテゴリタブ（44px高・水平スクロール）で絞り込み
  - タップでカートに+1。press演出＋カート側の該当行が一瞬ハイライト
- **カート（右ペイン・`--bg-card`・常時表示）**
  - 行: 商品名16px + ステッパー（− / 数量 / ＋。**ボタン各44×44px**・数量20px/600タブラー）+ 行金額
  - 下部: 合計（24px/700）→ 支払方法ボタン（現金/カード等・56px）→ **「会計する」プライマリ64px** → 成功演出 → 「領収書を発行」の後続導線（既存即売領収書フロー）
  - 空カート時: 「商品をタップして追加」16px muted。会計ボタン無効
- **部屋付け**は対象外（部屋付け物販は既存の予約詳細=PC業務。POSタブは即売専用とし、UIを最短動線に保つ）→ **要ユーザー確認**: 対面で「部屋につけておいて」と言われるシーンが多いなら、カートに「部屋付け」切替（在室予約の検索選択）を追加する。初期リリースは即売専用を推奨

### 4.6 部屋状況タブ（閲覧専用）

既存 RoomIndicator の閲覧専用ミニマル版。編集モード・D&Dは搭載しない。

- フロア別セクション（見出し「3F」16px/700 + 罫線）
- 部屋カード: 96×88px・角丸 `--radius-md`・グリッド（`auto-fill, minmax(96px, 1fr)`・gap 12px）
  - 上段: 部屋番号 18px/700、下段: 状態ラベル 12px
  - 状態の色分け（左ボーダー3px方式・Patina準拠）: 在室=`--accent-blue`（ブラス）、空室清掃済=`--accent-green`、未清掃/清掃中=`--accent-yellow`、点検中=`--accent-teal`、本日CI予定=輪郭ドット、故障・売止=`--text-muted`+斜線地
  - ゲスト滞在中は部屋カード内にゲスト名12pxを表示（タップで予約詳細は開かない。閲覧専用を徹底し誤操作を防ぐ）
- 凡例をコンテンツ上部に固定表示（12px・ドット付き）
- ポーリング更新（10秒）

## 6. 縦持ち（1032×1376）対応

- タブバー・ヘッダーはそのまま（幅1032pxで4タブ+バッジは収まる）
- CI確認・CO精算の**横2ペインは縦1カラムに積む**（`@media (max-width: 1080px)`。左ペイン→右ペインの順）。下部アクションバーは固定のまま
- POSは商品グリッドが上・カートが下部シート（高さ40%・スクロール）に切替
- 本日ボードのカードは1列のまま（変更なし）

## 7. インタラクション詳細

- press演出: すべてのタップ可能要素に `:active { transform: scale(0.98); }` + 面色。transition 150ms（`--transition-fast`）
- 画面遷移: ボード→詳細は右からのスライドイン200ms（`prefers-reduced-motion`で無効化）
- ポーリング: 一覧画面のみ。**詳細画面・ダイアログ表示中・入金入力中は停止**（編集中の上書き防止。計画書リスク5）
- pointerup後100msのclick抑制は不要（このモードにドラッグ操作は無い）
- スクロール領域に `overscroll-behavior: contain`
- 入力要素はすべて `font-size: 16px` 以上（iOSズーム回避）
- 日付・数値入力はこのモードには原則登場させない（入金金額のみ。`inputmode="numeric"`）
