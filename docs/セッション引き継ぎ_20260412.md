## 実装した機能

### 1. 予約統合機能

OTAが連泊を1泊ずつ別予約で通知するケースに対応。スタッフが手動で複数予約を1つにまとめる。

**設計思想:**
- `reservations` の1レコード = 1滞在（通常予約も統合予約も同一構造）
- PMS内部の予約番号は `reservations.id`（主キー）
- 統合後は `reservation_no = NULL`, `channel = NULL`（特定OTAに属さない）
- 元のOTA予約番号は全て `reservation_sources` テーブルに保持（親自身の分も含む）
- 通常予約は `reservation_no`/`channel` にOTA情報がそのまま入り、`reservation_sources` を持たない
- チャネル混在OK（同一ゲストが異なるOTAから1泊ずつ予約するケースがあるため）

**DB変更:**
- `reservation_sources` テーブル新規作成（マイグレーション 010）
- `reservations.status` に `merged` 追加
- `reservations.channel` を NOT NULL → NULL許可に変更

**API:**
- `POST /api/v1/reservations/merge` — 日程連続性チェック → 全OTA情報をsourcesに記録 → 親のchannel/reservation_noをNULLに → 子をmerged化
- `POST /api/v1/reservations/:id/split` — 統合予約を指定日で前半/後半に2分割（unmergeではない）

**TlImportService改修:**
- `findExistingReservation()` で `reservation_sources` も検索
- source経由でヒット時: 取消/日程変更 → `merge_alert`イベント記録、金額変更のみ → charges自動更新

**既存ロジック対応:**
- 予約一覧: merged除外、ステータス件数から除外
- 予約番号検索: `reservation_sources.reservation_no` 経由で親予約もヒット
- 予約一覧UI: `channel=NULL` は紫色の「統合」バッジ、予約番号は `#ID` 形式
- 予約詳細UI: トップバーに「統合」バッジ、右カラムに統合元予約カード（OTA一覧+分割ボタン）

### 2. 名寄せ機能（ゲスト紐付け・マージ）

**API（3本）:**
- `GET /api/v1/guests/match` — マッチング候補検索（名前・電話番号で類似ゲスト検索）
- `POST /api/v1/reservations/:id/link-guest` — 予約→ゲスト紐付け変更
- `POST /api/v1/guests/:id/merge` — ゲストマージ（複数ゲストを1つに統合）

**UI入口（3箇所）:**
- **予約一覧**: ゲスト名横の🔍ボタン（guest_match_status=pending時のみ）→ 候補検索ダイアログ

- **顧客管理**: ゲスト詳細の「重複ゲストをマージ」→ GuestMergeCard

**電話番号検索:**
- ハイフン・スペース・括弧・+を除去して正規化比較
- `090-1234-5678` = `09012345678` = `(+81)90-1234-5678` が相互にマッチ
- DB保存時は原本のまま保持（OTAフォーマットが多様なため正規化しない）

**予約一覧の検索:**
- ゲスト名・予約番号に加えて電話番号でも検索可能に

### 8. アサインボードから予約統合

予約一覧画面のみにあった統合導線を、アサインボードからも使えるようにした。今後さらに他画面からも導線を追加する予定の第一弾。

**操作フロー:**
1. ヘッダーの「統合」ボタンを押す → 統合選択モードに入る
2. ボード上のバー（SingleBar / LinkedBar）や右パネルの未アサインカードをクリックで選択/解除
3. 2件以上選択すると画面下部にフローティングアクションバー表示
4. 「統合実行」→ 各予約の詳細をGET取得 → ステータスチェック → 確認ダイアログ → `POST /reservations/merge`
5. ESCキーまたはトグルボタンで統合モード終了

**UIの挙動:**
- 統合モード中はカーソルが十字に変化（選択可能を示す）
- 選択されたバーは黄色アウトライン（`#fbbf24`）＋発光シャドウでハイライト
- 統合モード中はドラッグ＆ドロップ無効、右クリックメニュー無効（操作の干渉防止）
- 統合トグルボタンはアクティブ時に黄色ハイライト（ReservationListPageの統合ボタンと同系色で統一）

**設計判断:**
- `mergeSelectedIds` は `reservation_id` の Set（merge APIが `reservation_ids` を要求するため。同一予約の複数アサインも自然に重複排除）
- 統合実行時に各予約を `GET /reservations/{id}` で個別取得する理由: AssignControllerのSELECTには `updated_at` が含まれておらず、楽観ロック（規約#16）に必要。統合は低頻度操作なので2-5件の追加リクエストは許容範囲
- 統合ロジックの共通フック化は今回見送り。まだ2画面目であり、3画面目の追加時に2画面のパターンを見て抽出する方が適切

**State追加:**
- `mergeMode` — 統合選択モードのON/OFF
- `mergeSelectedIds` — 選択された reservation_id の Set

### 9. merge APIバグ修正（3件）

統合実行時に発生したバグを修正。

**バグ1: reservation_sources ユニークキー重複**
- 原因: `uk_channel_resno (channel, reservation_no)` が厳しすぎた。分割→再統合で同じ予約番号を別の統合先に登録できなかった
- 修正: ユニークキーを `(reservation_id, channel, reservation_no)` に変更（マイグレーション011）
- mergeロジックで子のsources付替え後に `(channel, reservation_no)` 重複を自動削除する処理を追加

**バグ2: 統合後にアサインのCO日が更新されない**
- 原因: 予約の `checkout_date` は延ばしていたが、親のアサインの `check_out_date` を更新する処理がなかった
- 修正: mergeトランザクション内にステップ6「親アサインのCO日拡張」を追加

**バグ3: 二重統合チェックが再統合を妨害**
- 原因: `original_reservation_id` でsourcesを検索する二重統合チェックが、分割で `confirmed` に戻った予約の再統合をブロック
- 修正: チェック削除。ステータスチェック（`confirmed` のみ許可）で十分

### 10. 統合解除機能（unmerge-source）

統合予約から特定のsource予約を外して独立予約に戻す機能。

**操作フロー:**
- 予約詳細の「統合元予約」カードで対象行をクリック → 確認ダイアログ → API呼び出し

**API:** `POST /api/v1/reservations/:id/unmerge-source`
- `source_id`: 解除するsourcesのID
- `updated_at`: 楽観ロック用

**処理内容:**
- 指定sourceの元予約を `confirmed` に復元（channel/reservation_no/日程/金額を戻す）
- 該当日程の charges を子に返却
- 残りsourcesの日程連続性をチェック → 連続しなくなる場合は統合を完全解除（全sourceを独立予約に復元）
- 残り1件でも完全解除
- 完全解除時はアサインも物理削除（CI前のアサイン操作 — CLAUDE.md準拠）

### セッション2 追加改修（後半）

### 19. 予約一覧のデフォルト表示を本日CI分に変更

初回アクセス時（URLパラメータなし）はCI日を本日に設定。運用時は当日分がデフォルト表示される方がフロントスタッフに使いやすい。

### 20. ステータス件数を日付フィルターに連動

ステータスチップ（全て/予約確定/CI中/CO済等）の件数が全件カウントだった問題を修正。日付・チャネル・検索条件に連動して期間内の件数を表示するように。

### 21. 日付フィルターの開始日・終了日の整合性

開始日を変更した際、終了日が開始日より前なら自動で開始日に揃える。終了日のカレンダーには `min` 属性で開始日より前を選択不可に。

### 22. 検索範囲トグル（期間内/全件）

検索窓の右端に「期間内」「全件」のトグルボタンを配置。
- **期間内**（デフォルト）: 日付フィルター連動で検索
- **全件**（オレンジ色）: 日付フィルターを無視して全予約から検索（過去の予約をゲスト名で探す用途）
- URLパラメータ `search_scope=all` で管理

### 23. CI日/CO日切替

日付フィルターの基準を「CI日」と「CO日」で切り替えるトグルボタンを追加。
- フロント: `date_type` URLパラメータ（`ci` or `co`）
- バックエンド: `date_type` パラメータで `checkin_date` / `checkout_date` を切替。ステータス件数SQLも連動

### 24. キャンセル予約のデフォルト非表示

キャンセル・ノーショー予約をデフォルト非表示に。ステータスチップの右端に「キャンセル表示」チェックボックスを配置。
- フロント: `show_cancelled` URLパラメータ、非表示時はキャンセルチップを薄く表示（dimmed）
- バックエンド: `hide_cancelled=1` パラメータで `status NOT IN ('cancelled', 'no_show')` フィルター

### 25. タイムラインにOTA操作日時を表示

`reservation_events.event_at` を `NOW()`（PMS取込時刻）→ OTA予約日時（`TravelAgencyBookingDate + Time`）に変更。
- `recordEvent()` に `$eventAt` パラメータ追加
- 全TLイベント（新規/変更/取消/統合アラート）の5箇所でOTA日時を渡す
- フロントのタイムラインラベルを `tl_new`→「新規予約」、`tl_modify`→「予約変更」、`tl_cancel`→「予約取消」に変換
