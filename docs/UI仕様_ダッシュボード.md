# ダッシュボード UI仕様書

UIモック（pms_dashboard.html）を参考にした画面仕様。

---

## 1. 画面の役割

ログイン直後のホーム画面。フロントスタッフが一日中見ている画面。アイドル時の基本画面。

- 必要な情報がパッと見える（数字のサマリー）
- 異常があればすぐ気づける（アラート集約）
- 1クリックで詳細画面に遷移できる（ハブとしての機能）

---

## 2. レイアウト構成

```
┌─ サイドナビ（全画面共通・折りたたみ） ─┬──────────────────────────────────────┐
│                                      │ ヘッダー                              │
│  ◉ ダッシュボード ← active            │  ホテル名 ── サマリーバー ── 時計/スタッフ │
│  ☰ 予約一覧                          ├──────────────────────────────────────┤
│  ▦ アサインボード                     │                                      │
│  ⊞ ルームインジケーター               │  ① アラートセクション                   │
│  ◫ 清掃管理                          │                                      │
│  ◈ 売上レポート 🔒                    │  ② CI/COリスト（2カラム横並び）          │
│  ⚙ 設定 🔒                           │                                      │
│                                      │  ③ 補助情報（明日プレビュー・TLログ）     │
│                                      │                                      │
└──────────────────────────────────────┴──────────────────────────────────────┘
```

## 3. ヘッダー（サマリーバー統合型）

サマリーカードを廃止し、ヘッダーバーに統合。全画面共通。

### 3-1. 上段

| 左 | 右 |
|----|-----|
| ホテル名（PATINA HOTEL OKINAWA） | リアルタイム時計 ・ スタッフ名 ・ ログアウトボタン |

### 3-2. 下段（サマリーバー）

4つのサマリー情報を横1行で表示。各項目の区切りに縦セパレータ。

| 項目 | 表示内容 | アクセントカラー | クリック遷移先 |
|------|---------|----------------|-------------|
| CI予定 | 件数 + 未CI件数（赤字） | 青 #3B82F6 | 予約一覧（本日CIフィルタ） |
| CO予定 | 件数 + 未CO件数（赤字） | 紫 #8B5CF6 | 予約一覧（本日COフィルタ） |
| 稼働率 | 在室数/全室数 + パーセンテージ | 緑 #22C55E | ルームインジケーター |
| 清掃 | 清掃中・点検待・Ready の件数 | ティール #14B8A6 | 清掃管理画面 |

各項目はホバーでハイライト。左端に3px幅の色付きアクセントバー。

**設計判断：** 元々サマリーカード4枚を配置していたが、ヘッダーに統合することで縦スペースを節約。アラートやCI/COリストがファーストビューに収まる。

## 4. アラートセクション

ダッシュボードで最も重要な部分。フロントが見落としてはいけない事項を集約。

### 4-1. ヘッダー

```
⚠️ 要対応  [3件]  ← 赤背景バッジ
```

### 4-2. アラート種別と表示

| 種別 | ドットカラー | ラベル背景 | アクション |
|------|------------|----------|----------|
| ゲスト未確定 | 🔴赤 | 赤系 | [確認する] → 予約詳細 |
| 未チェックイン | 🟡黄 | 黄系 | [ノーショー処理] / [確認済み（様子見）] |
| 連結予約の穴 | 🟡黄 | 黄系 | [確認する] → 連結予約詳細 |
| 定員超過アサイン | 🟡黄 | 黄系 | [確認する] → アサインボード |
| 部屋タイプ不一致 | 🟡黄 | 黄系 | [確認する] → アサインボード |
| 未アサイン（CI前日） | 🟡黄 | 黄系 | [確認する] → アサインボード |
| TL取込エラー | 🔴赤 | 赤系 | [確認する] → TL取込ログ詳細 |

### 4-3. 各アラート行の構成

```
[ドット] [ラベルバッジ] 説明テキスト                      [アクションボタン]
         サブテキスト（候補情報・OTA・部屋等）
```

### 4-4. 0件の場合

```
✅ 対応事項はありません
```

セクション自体を非表示にしない（確認したが0件だったことを示すため）。

## 5. 本日のCI/COリスト

2カラム横並び。

### 5-1. リスト行の構成

```
[✓/○] [OTAドット] ゲスト名          部屋 タイプ    [ステータスバッジ]
```

- ✓（緑）= CI済/CO済、○（グレー）= 未CI/未CO
- ステータスバッジ：「14:00 CI済」（緑背景）/ 「未CI」（黄背景）

### 5-2. 件数制限

- 最大5件表示
- 残りは「全件表示 →」リンクで予約一覧（日付フィルタ適用済み）に遷移
- 行クリックで予約詳細画面へ遷移

---

## 6. 補助情報（2カラム横並び）

### 6-1. 明日のプレビュー

- CI件数・CO件数（大きい数字表示）
- 稼働予測（プログレスバー + パーセンテージ）
  - 80%超はイエロー、それ以下はブルー
- クリックで予約一覧（明日フィルタ適用）

### 6-2. TL取込ログ

- 直近5件を表示
- 正常: 時刻 + OTAカラードット + OTA名 + 予約番号
- エラー: 赤背景行 + ❌ エラー表示
- 緑点滅ドット（TL接続の正常性インジケーター）

---

## 7. リアルタイム更新（ポーリング間隔）

| セクション | 間隔 |
|-----------|------|
| サマリーバー | 10秒 |
| アラート | 10秒 |
| CI/COリスト | 10秒 |
| TL取込ログ | 30秒 |
| 明日のプレビュー | 60秒 |

## 8. サイドナビ（全画面共通）

折りたたみ可能（デフォルト展開）。

### 8-1. メニュー項目

| アイコン | ラベル | 権限 |
|---------|-------|------|
| ◉ | ダッシュボード | 全員 |
| ☰ | 予約一覧 | 全員 |
| ▦ | アサインボード | 全員 |
| ⊞ | ルームインジケーター | 全員 |
| ◫ | 清掃管理 | 全員 |
| ◈ | 売上レポート | report.view権限 🔒 |
| ⚙ | 設定 | 設定系権限 🔒 |

### 8-2. 表示仕様

- アクティブ項目: 青ハイライト + 右ボーダー
- 権限不足項目: 🔒アイコン表示
- 折りたたみ時: アイコンのみ表示（56px幅）
- 展開時: アイコン + ラベル（220px幅）
- フッター: バージョン情報
