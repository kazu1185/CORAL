# アサインボード UI仕様書

UIモック（pms_assign_board.html）を参考にした画面仕様。

---

## 1. 画面の役割

横軸＝日付、縦軸＝部屋のガントチャート型画面。期間全体を俯瞰して部屋割りを管理する。**アサイン操作の本命画面。**

現行PMSではドラッグ操作ができなかったが、新PMSではドラッグで全操作ができることが必須要件。

---

## 2. レイアウト構成

```
┌─ サイドナビ(折畳) ─┬──────────────────────────────────────────────────┐
│                    │ ヘッダー（タイトル + 日付ナビ + 時計）               │
│  ▦ アサインボード   ├──────────────────────────────────┬──── [▶] ─────┤
│    ← active        │                                  │ 右パネル       │
│                    │  部屋列(固定) │ ガントチャート本体  │ （折りたたみ式）│
│                    │               │                  │               │
│                    │  201 シングル  │ ██ バー ██        │ サマリー       │
│                    │  202 シングル  │    ██ バー        │ 未アサイン     │
│                    │  ...          │                  │ アラート       │
│                    │               │                  │ OTA凡例       │
└────────────────────┴──────────────────────────────────┴───────────────┘
```

## 3. ヘッダー

| 左側 | 右側 |
|------|------|
| 画面タイトル「▦ アサインボード」 | リアルタイム時計 ・ スタッフ名 |
| 日付ナビ（後述） | |

### 3-1. 日付ナビゲーション

```
[◀ 1週] [◀] [4/9 〜 4/18] [▶] [1週 ▶] [今日]
```

- ◀▶ で1日単位移動
- ◀1週 / 1週▶ で7日単位移動
- 「今日」ボタンで今日の1日前が左端に来る位置にリセット

## 4. ガントチャート本体

### 4-1. 表示期間・セル幅

| 項目 | 値 |
|------|-----|
| 表示日数 | 10日間 |
| セル幅 | 120px（CSS変数 `--cw`） |
| 今日の左 | 1日分（今日が2列目に来る） |

### 4-2. 部屋列（左固定）

- 横スクロールしても常に表示（`position: sticky`）
- フロア別グループ表示（2F / 3F / 4F / 5F / 6F）
- 各行: 部屋番号（太字） + 部屋タイプ（グレー）
- 行高: 44px
- フロアヘッダー: 24px（グレー背景）

### 4-3. 日付ヘッダー（上部固定）

- 縦スクロールしても常に表示（`position: sticky`）
- 表示: 月/日 + 曜日
- 今日: ブルー背景 + 太字
- 土日: 赤色テキスト
- 今日列のセル背景: 薄いブルー（全行に適用）

### 4-4. グリッドセル

- 各セル120px幅
- 薄いボーダーで区切り
- 今日列はうっすらブルー背景

---

## 5. バー表示

### 5-1. 基本バー

- OTAカラーの角丸バー（border-radius: 6px）
- 高さ32px（セル高44pxの中で上下6pxマージン）
- 表示内容: ゲスト名 + 泊数

### 5-2. 泊数に応じた表示の可変化

| 泊数 | バー幅 | 表示内容 |
|------|--------|---------|
| 1泊 | 116px | 姓のみ（例:「山田」）。名前が長い場合は泊数のみ |
| 2泊 | 236px | フルネーム + 泊数 |
| 3泊以上 | 356px〜 | フルネーム + 泊数（十分な余白） |

**1泊バーでは名前が収まらない場合がある。ホバーのツールチップで詳細を確認する設計。**

### 5-3. 特殊バー表示

| 状態 | CSSクラス | 表示方法 |
|------|----------|---------|
| ノーショー | `.noshow` | ストライプ柄（45度斜線） + 半透明（opacity: 0.5） |
| 途中退室 | `.early-out` | 破線ボーダー（dashed） + 薄い背景 |
| 分割予約 | `.linked` | 上辺に紫ボーダー（3px solid #8B5CF6） |
| 連結予約の穴 | `.gap` | 赤い点滅枠（animation: blink 1s infinite） + 赤ストライプ背景 |
| 途中部屋移動（移動元） | `.move-from` | バー右端に「→」マーク |
| 途中部屋移動（移動先） | `.move-to` | バー左端に「←」マーク |

### 5-4. 部屋移動の表示ルール

- 移動元バーと移動先バーは別の部屋行に表示
- 移動情報（「→304へ」「303より→」）はバー上には表示しない（スペース不足のため）
- ツールチップに「移動: →304へ」として表示

## 6. ツールチップ

バーホバーで表示される黒背景の情報パネル。

```
┌─────────────────────┐
│ 山田 太郎             │
│ OTA        電話       │
│ CI         4/10      │
│ CO         4/13（3泊）│
│ 部屋       303 ツイン  │
│ ステータス  confirmed  │
│ 移動       →304へ     │  ← 移動がある場合のみ
└─────────────────────┘
```

- 穴（gap）バーにはツールチップを表示しない
- バーの下に8px空けて表示
- 画面右端に近い場合は位置を調整

---

## 7. ドラッグ&ドロップ操作（実装時）

UIモックでは表示のみ。実装時に以下のドラッグ操作を実装する。

| 操作 | 動作 |
|------|------|
| 未アサイン予約を部屋行にドラッグ | アサイン |
| バーを別の部屋行にドラッグ | 部屋移動 |
| バーの途中で分割→後半を別室にドラッグ | 途中部屋移動 |
| バーの右端をドラッグ | 延泊・短縮 |
| バークリック | 予約詳細画面に遷移 |

### 7-1. アサイン時の警告

ドラッグでアサインした際に定員超過・部屋タイプ不一致があれば警告ダイアログを表示。ただしブロックはしない（「アサインする」で続行可能）。

## 8. 右パネル（折りたたみ式）

### 8-1. トグルボタン

- ガントと右パネルの境界に ▶/◀ ボタン
- クリックで右パネルの展開/折りたたみ
- 折りたたみ時: パネル幅0、ボタンが右端に移動
- 展開時: パネル幅280px

### 8-2. パネル内セクション

**① 本日のサマリー**
| 項目 | 表示 |
|------|------|
| CI予定 | 件数 |
| CO予定 | 件数 |
| 稼働率 | パーセンテージ (在室/全室) |
| ADR | 金額 |

**② 未アサイン予約リスト**
- 黄色背景のカード型
- ゲスト名 + OTAドット + 日程 + 室タイプ + 泊数
- ドラッグ可能（実装時にガントへのドロップでアサイン）

**③ 要対応アラート**
- ダッシュボードと同じアラート情報のサマリー

**④ OTA凡例 + 特殊バー凡例**
- OTAカラー7色の凡例
- ノーショー / 途中退室 / 分割予約 / 穴 の表示サンプル
