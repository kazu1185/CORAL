```

**作業内容:**

```

PIN変更ダイアログ:
  - 現在のPIN入力 → 新しいPIN入力 → 確認入力 → 変更完了
  - PUT /api/v1/auth/pin

デザイン:
  - 中央寄せのカード型
  - ホテル名「PATINA HOTEL OKINAWA」をカード上部に表示
  - 「PMS ログイン」のサブタイトル

- PIN入力で認証成功 → ダッシュボードに遷移
- PIN誤り → エラーメッセージ表示

**アラート種別の判定ロジック（SQLクエリのWHERE条件）:**

| 種別 | 判定条件 |
|------|---------|
| ゲスト未確定 | guest_match_status = 'pending' AND status = 'confirmed' AND checkin_date <= 今日+1 |
| 未チェックイン | checkin_date < 今日 AND status = 'confirmed' |
| 連結穴 | guest_links に status = 'gap' がある |
| 定員超過 | room_assignments JOIN rooms/room_types で adult_count > max_adults |
| タイプ不一致 | room_assignments の room.room_type_id != reservations.room_type |
| 未アサイン（CI前日） | checkin_date = 明日 AND room_assignments なし |
| TL取込エラー | tl_import_logs に parse_status = 'error' かつ直近24時間以内 |

**フロントエンド:**
- ポーリング間隔: サマリー/アラート/CI/COリスト → 10秒、TLログ → 30秒、明日プレビュー → 60秒
- CI/COリストは最大5件表示 + 「全件表示→」リンク
- アラート0件時「✅ 対応事項はありません」表示
- 各セクションからの1クリック遷移

**完了条件:**

## 全画面共通ルール

- **デフォルトテーマ: ライト（白背景）。** ダークモードはCSS変数切替で対応
- **フォント: Noto Sans JP** (Google Fonts CDNから読み込み)
- **ポーリング: WebSocket不使用。** setInterval + fetch で実装
- **物理DELETE禁止。** 全てステータス変更による論理削除
- **updated_at / updated_by を持つテーブルは更新時に必ずセットすること**
- **TLリンカーンへの書き込みコードは絶対に書かない**（受動的PMSの原則）
- **staff_activity_logs への記録は重要操作のみ**（閲覧操作は記録しない）
