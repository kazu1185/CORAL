### channelsテーブル構造

```sql
CREATE TABLE channels (
    id INT AUTO_INCREMENT PRIMARY KEY,
    channel_code VARCHAR(20) NOT NULL UNIQUE,   -- 不変キー
    channel_name VARCHAR(50) NOT NULL,          -- 表示名
    color VARCHAR(7) NOT NULL DEFAULT '#6B7280',-- バッジカラー
    channel_type ENUM('ota','manual') NOT NULL, -- ota=TL経由, manual=手動
    tl_match_patterns VARCHAR(200) NULL,        -- TL CompanyName部分一致パターン（カンマ区切り）
    sort_order INT NOT NULL DEFAULT 0,
    is_active TINYINT(1) NOT NULL DEFAULT 1,
    ...
);
```

### 初期データ（15件）
OTA: jalan, rakuten, booking, agoda, expedia, ikyu, jtb, ana, jal, skyticket, other
手動: direct, phone, walkin, corporate

### 未登録OTAアラートフロー
1. TL XMLの`SalesOfficeCompanyName` → channelsテーブルの`tl_match_patterns`で照合
2. マッチしない → `channel='other'`で予約登録 + `unknown_channel`イベント記録（`detail`にcompany_name保存）
3. ダッシュボードに「未登録OTAの予約: XXX（N件）」黄色アラート表示（30日以内）
4. アラートクリック → `/settings/channels` に遷移
5. 新チャネル登録後「other予約を一括変更」ボタンで既存予約のchannelを更新可能

### TlXmlParserの変更方針
- `OTA_CHANNEL_MAP` 定数はフォールバック用として`OTA_CHANNEL_MAP_FALLBACK`にリネーム
- `setChannelMap(array $channels)` メソッドでDB由来のマップを外部注入
- `TlImportService`のコンストラクタでDBから読み込んでパーサーに注入
- DB未接続時（CLIテスト等）はフォールバック定数を自動使用

### 設定画面のAPI

```
GET  /api/v1/master/channels          — 一覧取得（?all=1で無効含む）
POST /api/v1/master/channels          — 新規作成
PUT  /api/v1/master/channels/:id      — 更新（channel_code変更不可）
POST /api/v1/master/channels/remap-other — other予約の一括チャネル更新
```

### 実装ファイル

| ファイル | 種別 | 内容 |

### Props

| Prop | 型 | 説明 |
|------|---|------|
| `mode` | `'single' \| 'range'` | 単日選択 or 範囲選択 |
| `value` | `string` or `{ from, to }` | 選択中の日付（YYYY-MM-DD） |
| `onChange` | `(value) => void` | 日付変更コールバック |
| `minDate?` | `string` | 選択可能な最小日付 |
| `maxDate?` | `string` | 選択可能な最大日付 |
| `highlightToday?` | `boolean` | 今日ハイライト（デフォルトtrue） |

### 範囲選択の操作
1. 1回目クリック → CI日設定（青丸）、CO日リセット
2. ホバー → CI〜マウス位置の範囲をプレビュー（薄青）
3. 2回目クリック → CO日設定（青丸）、範囲確定
4. カレンダー下部に「2026-04-20 → 2026-04-23（3泊）」のテキスト表示

### ビジュアル
- 今日: 青い丸枠
- 選択端点（CI/CO）: 青背景・白文字
- 範囲内: 薄青背景
- 日曜: 赤文字、土曜: 青文字
- 前月/翌月の日: 薄グレー

### 手動予約入力での統合
- カレンダーの右側に泊数・部屋タイプ・プラン・人数フィールドを横並び配置
- カレンダーで範囲選択 ↔ 泊数inputが双方向同期
- 範囲変更で料金明細テーブルが自動更新

## 8. 次セッションで実装するタスク: 到着予定時刻

### 概要
TL電文の `tl_other_info` に到着時刻情報が含まれている。OTAごとにフォーマットが異なる:
- **Booking.com**: `Approximate time of arrival: between 15:00 and 16:00`
- **じゃらん**: フライト情報（`2026-05-01,ANA0089,羽田-石垣,08:30-11:30`）
- **その他**: チェックイン時間帯の記載

### 実装内容
1. **TlXmlParser** — `tl_other_info` から到着時刻を抽出するパーサーを追加
   - Booking.com: `Approximate time of arrival: between HH:MM and HH:MM` → 後者のHH:MMを採用
   - じゃらん: フライト到着時刻 + 移動時間マージン
   - 正規表現で各フォーマットを順次試行
2. **TlImportService** — パース結果を `estimated_arrival` カラムに保存
3. **予約詳細画面** — 到着予定時刻の表示+編集機能を追加
   - 自作の時刻ピッカー（ブラウザ標準の`<input type="time">`は使わない）
   - 15分刻みのドロップダウン or ダイヤル式
   - 編集 → `PUT /reservations/:id` で保存
4. **ルームインジケーター** — 既にカードに表示する実装は済み（データが入ればすぐ表示される）
