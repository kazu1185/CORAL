Phase 10 で基盤実装後、UI/UX改修とビジネスロジック整備を実施した記録。

## 画面構成

```
┌─────────────────────────────────────────────────────────┐
│ ≪1週間前 | ‹1日前 | 今日 | 1日後› | 1週間後≫   期間表示  [未アサイン 23] │
├───────────────────────────────────────────┬─────────────┤
│                                           │ 未アサイン   │
│  部屋 │ 9 │10 │11 │12 │13 │ ...         │ パネル       │
│  201  │████████│   │██████│              │（オーバーレイ）│
│  202  │   │████████████│                 │              │
│  ...  │                                  │              │
│                ガントチャート              │              │
│           （12日間表示・横幅いっぱい）      │              │
└───────────────────────────────────────────┴─────────────┘
```

- 表示期間: 12日間（前日〜10日後）
- 未アサインパネル: 右側オーバーレイ（チャートに被せる）、折りたたみ可能、状態はlocalStorage保存
- バー幅: `calc(泊数 * 100% - 2px)` でセルに対する相対指定（リサイズ追従）
- 日付処理: `parseLocal()` でローカル時間統一（UTC変換によるずれ防止）

## バー表示

### 通常バー
- OTAカラーで背景色
- ゲスト名 + VIPバッジ(V) + CI済みバッジ(CI)
- ホバー400msでツールチップ（ゲスト名・OTA・予約番号・日程・人数・金額・部屋タイプ）

### 連結バー（分割予約）
- 同じ `link_group_id` のアサインを1本のバーに結合
- 内部をflex分割、各セグメントにOTA背景色、白い区切り線(2px)
- ゲスト名は先頭セグメントのみ表示
- ホバーで全セグメントのOTAを表示

### CI済みバー
- 白ボーダー + 「CI」バッジ
- ドラッグ不可、右クリックメニュー非表示、リサイズハンドル非表示

## 操作一覧

### CI前（confirmed）の操作

| 操作 | 方法 | 動作 |
|------|------|------|
| 新規アサイン | 未アサインカードをD&Dでセルにドロップ | 確認ダイアログ → アサイン作成 |
| 部屋移動 | バーをD&Dで別の行にドロップ | 確認ダイアログ → 旧アサイン物理削除 + 新アサイン作成 |
| 部屋移動 | 右クリック → 部屋移動 | ダイアログで移動先選択 → 同上 |
| 途中移動 | 右クリック → 途中移動 | ダイアログで日付+移動先選択 → 元アサイン短縮 + 新アサイン作成 |
| アサイン取消 | 右クリック → アサイン取消 | 確認ダイアログ → 物理削除 |
| 延泊 | バー右端を右にドラッグ | 確認ダイアログ → アサインCO日延長 + 予約CO日/nights連動更新 + 売上明細自動生成（金額0） |
| 短縮 | バー右端を左にドラッグ | 確認ダイアログ → アサインCO日のみ短縮（予約データ・明細には触らない） |
| 予約プレビュー | バーをクリック | モーダルで予約詳細表示（画面遷移なし） |
| 予約プレビュー | 未アサインカードをクリック | 同上 |

### CI後（checked_in）の操作

| 操作 | 方法 | 動作 |
|------|------|------|
| アサインボードから | 全操作 | **ブロック**（ドラッグ不可・右クリック非表示） |
| 部屋移動 | 予約詳細画面 → 部屋移動ボタン | ダイアログで移動先選択 → released履歴を残す + 新アサイン作成 |
| 途中退室 | 予約詳細画面 → チェックアウト | CO処理フローで明細をキャンセルポリシーに従い処理 |

## CI前 vs CI後の設計思想

| | CI前（アサイン作業） | CI後（実際の滞在） |
|---|---|---|
| 目的 | 割り付け調整 | お客様の実際の移動記録 |
| released履歴 | **残さない**（物理削除/上書き） | **残す**（移動記録として保持） |
| 予約データ連動 | 延泊時のみ連動 | CO処理で連動 |
| 操作場所 | アサインボード | 予約詳細画面 |

## 延泊フロー

```
アサインボードで延泊（右端ドラッグ）
  ↓
確認ダイアログ「○○のCO日を 04/12 → 04/14 に延泊しますか？」
  ↓ OK
① room_assignments.check_out_date を更新
② reservations.checkout_date, nights を連動更新
③ reservation_charges に延泊分の行を自動追加（金額0、摘要「延泊（料金未設定）」）
  ↓
ダッシュボードに黄色アラート「延泊料金未設定 — ○○（N泊分）」
  ↓
予約詳細の売上明細で延泊行に料金入力 → 「設定」ボタンで保存
  ↓
予約合計金額を再計算（active分の明細合計）
```

---

## 短縮の注意事項

- アサインボードの短縮は**アサインの日程だけ**変更する
- 予約CO日・売上明細には**触らない**
- 理由: TLから流れてきた予約のCO日を勝手に変えるとOTAデータとの整合性が崩れる
- 実際の途中退室は予約詳細画面のCO処理で行い、キャンセルポリシーに従って明細を処理する（cancel_fee / waived / そのまま）

## APIエンドポイント

| メソッド | パス | 用途 | CI前 | CI後 |
|---------|------|------|------|------|
| GET | /api/v1/assigns | 一覧（期間指定） | ○ | ○ |
| POST | /api/v1/assigns | 新規アサイン | ○ | ✕(403) |
| PUT | /api/v1/assigns/:id | 日程変更（延泊/短縮） | ○ | ✕(403) |
| DELETE | /api/v1/assigns/:id | アサイン取消（物理削除） | ○ | ✕(403) |
| POST | /api/v1/assigns/:id/move | 部屋移動 | ○（物理削除+新規） | source=detail のみ許可（released+新規） |
| POST | /api/v1/assigns/:id/split | 途中移動 | ○ | ○ |
