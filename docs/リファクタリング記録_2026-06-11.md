## フェーズA: コーディング規約違反の修正（フロントエンド）

### 規約 #2（日付処理）違反の解消
- `Math.round((new Date(co) - new Date(ci)) / 86400000)` 形式の泊数計算が
  ReservationDetailPage.jsx に7箇所 → すべて `dayDiff()` に統一
- `new Date(...).toISOString().slice(0, 10)` → `addDays()` に置換
- `utils/date.js` に3関数を追加:
  - `todayStr()` — 今日の 'YYYY-MM-DD'（`fmt(new Date())` の散在を集約）
  - `addDays(d, days)` — 日付文字列/Dateに日数加算して文字列を返す
  - `fmtDateTime(s, withYear)` — MySQL DATETIME → 表示用。`new Date(文字列)` の
    パースはブラウザ実装依存のため文字列操作のみで変換する設計
- AssignBoardPage / RoomInventoryPage の「今日±N日」の手書き計算ブロックを `addDays()` で簡潔化

### 規約 #15（共通パーツ使用）違反の解消
- `utils/constants.js` に追加:
  - `RESERVATION_STATUS_MINI` — GroupReservationPage のローカル定義を移設（表示文言は不変）
  - `MANUAL_CHANNELS` — ReservationCreatePage のローカル定義を移設

- StaffSettings の `ROLE_LABELS` 等、**1ファイルでしか使わないラベルは移設しない**と判断
  （重複がないため。むやみに constants.js を肥大化させない）
