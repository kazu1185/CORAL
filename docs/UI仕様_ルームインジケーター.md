# ルームインジケーター UI仕様書

UIモック（pms_room_indicator.html）を参考にした画面仕様。

---

## 1. 画面の役割

今日の部屋状態をパッと見る画面。CI/CO操作の入口。

- 全室の「今この瞬間の状態」をカード型グリッドで一覧表示
- カードクリックで予約詳細画面に遷移
- **この画面でアサイン操作はしない**（アサインはアサインボードで行う）
- 日付ベースではなく状態ベースで判定（営業日更新の概念は不要）

## 4. フロア別グリッド

### 4-1. フロアヘッダー

```
2F ─ シングル
```

- フロア番号 + 主な部屋タイプをラベル表示
- グレー文字、大文字、レターspacingで控えめに

### 4-2. カードグリッド

- レスポンシブ: `grid-template-columns: repeat(auto-fill, minmax(240px, 1fr))`
- カード間: 12px gap
- 1フロアごとに `margin-bottom: 24px`

### 4-3. フロア・部屋の配置はマスタ駆動

**フロアの順序、各フロアに属する部屋、部屋タイプ、部屋の並び順はすべて部屋マスタ（rooms / room_types テーブル）から動的に生成する。** HTMLにハードコードしない。

- 部屋の追加・削除・タイプ変更・フロア移動は管理画面の部屋マスタから行う
- フロアヘッダーのラベル（「2F ─ シングル」等）は、そのフロアに属する部屋タイプの代表名を自動生成する（1種類なら単一名、複数種類なら「ダブル / ラージ」のようにスラッシュ区切り）
- フロア内の部屋の表示順は `rooms.sort_order` カラムで制御
- フロアの表示順は `rooms.floor` カラムの昇順

**UIモックでは2F〜6Fの24室で固定表示しているが、実装時はマスタデータから動的にレンダリングすること。**

## 5. カード状態パターン（5種類）

### 5-1. 状態判定ロジック

```
各部屋に対して「今この瞬間の状態」を表示：

① room_assignmentsでactiveな予約がある
   → 在室中カード
   → さらに checkout_date = 今日 なら「CO予定」バッジ付き

② activeはないが、本日CI予定のconfirmed予約がある
   → CI予定カード

③ activeはないが、過去日CI予定のconfirmed予約がある
   → 未CIカード（赤・アラート扱い）

④ どの予約にも紐付いていない
   → 空室カード（清掃ステータス表示）
```

## 9. カードの振る舞い

- ホバー: 上に2px浮き上がり + 影を強める
- クリック: 予約詳細画面に遷移（空室カードはクリック不可 or 無反応）
- リアルタイム更新: 10秒間隔ポーリング（CI/CO完了で即座にカード状態が変わる）

---

## 10. 現行PMSとの違い（自前開発の動機）

現行PMSのルームインジケーターには最低限の情報（部屋番号・タイプ・泊数・ゲスト名）しかない。新PMSでは以下を追加表示する：

- リピーターバッジ（宿泊回数）
- VIPバッジ
- ゲストメモの有無
- 連結予約バッジ
- 支払方法
- 人数（大人/子供）
- 清掃ステータス（空室時）
